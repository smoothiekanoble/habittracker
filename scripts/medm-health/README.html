MedM Health Adapter Contract

This directory contains source-adapter work for MedM body-weight CSV exports. It intentionally does not define database schema, apply Supabase migrations, or build application UI.

Command

node scripts/medm-health/parse-medm-weight-csv.mjs scripts/medm-health/fixtures/synthetic-medm-weight.csv

Output Shape

{
  "date": "YYYY-MM-DD",
  "metric_type": "body_weight",
  "value": 181.8,
  "unit": "lb",
  "source": "medm_health",
  "source_detail": "Synthetic Scale; MedM Health CSV; time 07:15",
  "source_record_id": "medm-synth-001"
}

Rules

  • Use only synthetic fixtures in Git.
  • Keep real exports under local-data/imports/medm-health/.
  • Use source_record_id when supplied by the export; otherwise use raw_hash for idempotency.
  • Connect to shared daily_metrics only after the metrics-platform branch lands.
