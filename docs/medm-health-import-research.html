MedM Health Body Weight Import Research

Feasibility review for importing body-weight readings from MedM Health or a MedM-connected scale into Habit Tracker.

Prepared June 7, 2026. Worktree: codex/medm-health-import.

Recommendation

MedM body weight ingestion looks feasible for v1 through user-controlled file export, with Apple Health as the best iPhone source of truth if MedM sync is already enabled. Start with CSV/XLSX-style file import for weight readings and avoid undocumented cloud APIs.

Options Considered

PathFit for v1Notes
MedM app exportBest practical first pathMedM describes Health Diary as supporting basic data export, PDF/XLSX reports for Premium, and export/sharing options. Source: MedM Health Diary.
Apple Health syncBest iPhone source of truthThe MedM manual says Apple Health sync supports Weight and that MedM exports Weight, BMI, lean body mass, and body fat percentage to Apple Health where supported. Source: MedM Health user manual.
Google Health Connect syncGood Android pathMedM documents Health Connect support for Android and import/export of weight among other metrics. Useful if the source phone is Android, less central for this iPhone-first project.
MedM cloud/APINot for personal v1MedM has platform/cloud offerings, but this spike did not find a simple personal OAuth/API path for Habit Tracker. Treat any API work as a later vendor-integration project.
Manual entryAcceptable fallbackIf exports are annoying, body weight is low-frequency enough that manual entry remains viable.

Best Practical Path

For the user’s current iPhone-first v1, make Apple Health the canonical phone-side health hub. Let MedM write weight to Apple Health when possible, then import a user-controlled export from Apple Health or from MedM’s own report/export flow. This avoids depending on MedM credentials inside Habit Tracker.

Prototype

This spike includes a synthetic CSV parser for weight readings:

node scripts/medm-health/parse-medm-weight-csv.mjs scripts/medm-health/fixtures/synthetic-medm-weight.csv

It emits normalized daily metric records:

{ "date": "2026-06-01", "metric_type": "body_weight", "value": 181.8, "unit": "lb", "source": "medm_health", "source_detail": "synthetic_weight_csv 07:15" }

Data Fields Available

For weight, MedM documents Body Mass / Weight plus body composition fields such as body fat percentage, BMI, muscle metrics, water metrics, basal metabolic rate, visceral fat, lean mass, and metabolic age. The first import should only support body_weight, then add BMI/body-fat once a real export format is inspected locally under local-data/.

Manual User Steps

  1. Pair the body-weight scale with MedM Health Diary or enter weight manually.
  2. Enable Apple Health sync for Weight if using Apple Health as the hub.
  3. Export a weight report/file from MedM or Apple Health.
  4. Place the file under local-data/imports/medm-health/.
  5. Run a local dry-run importer and review normalized rows before uploading.

Risks and Limitations

  • Actual MedM export headers need to be inspected with a real local file before final parser hardening.
  • Premium subscription may affect available report/export and ecosystem sync options.
  • Body composition fields vary by scale and may not map cleanly across Apple Health, MedM, and Habit Tracker.
  • Do not commit real weight exports; keep them under gitignored local-data/.

Next Implementation Steps

  2. Adapt the parser to the actual headers while keeping the synthetic fixture in git.
  3. Add dry-run validation: duplicate detection, units, impossible values, and date parsing.
  4. Insert into shared daily metrics storage only after schema review.
