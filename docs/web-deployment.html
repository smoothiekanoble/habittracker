Web Deployment Checklist

This is the v1 shipping path: deploy the Next.js app in web/, keep Supabase as the data store, and install the deployed site on iPhone from Safari with Add to Home Screen.

Updated: 2026-05-15 Target: Vercel Repo root: C:\dev\personal\habittracker App root: web

Vercel Project

  1. Import git@github.com:smoothiekanoble/habittracker.git into Vercel.
  2. Set the Vercel project root directory to web.
  3. Keep the detected framework as Next.js.
  4. Use the default commands unless Vercel asks for them explicitly: install command npm install, build command npm run build, and framework-default output directory.
  5. Add NEXT_PUBLIC_SUPABASE_URL and NEXT_PUBLIC_SUPABASE_ANON_KEY for Production and Preview.
  6. Do not set NEXT_PUBLIC_AUTH_REDIRECT_ORIGIN in Vercel unless intentionally forcing OAuth to a specific origin. It is only needed for local LAN testing, where the phone must return to the laptop's IP address.

The app depends on @habittracker/shared via file:../shared. With Vercel's monorepo root directory set to web, the repository checkout should still include shared/. If Vercel reports that ../shared cannot be found, change the project root to the repository root and set the build command to cd web && npm install && npm run build.

Windows CLI Note

On this Windows machine, call the CLI as vercel.cmd instead of vercel if PowerShell blocks vercel.ps1 because script execution is disabled.

Supabase Auth URL Configuration

In Supabase Dashboard -> Authentication -> URL Configuration:

  1. Set Site URL to the final production origin, for example https://habittracker.example.com.
  2. Add the exact production callback URL: https://habittracker.example.com/auth/callback.
  3. Keep local development allowed if you still test locally: http://localhost:3000/auth/callback, http://127.0.0.1:3000/auth/callback, and http://<PC_LAN_IP>:3000/auth/callback for iPhone-over-Wi-Fi testing.
  4. For Vercel preview deployments, add a Vercel preview wildcard only if you plan to sign in on preview URLs: https://*-<team-or-account-slug>.vercel.app/**.

Supabase recommends exact redirect paths for production. Wildcards are useful for previews and local development, but the production callback should be explicit.

Google OAuth Provider

In the Google Cloud OAuth client used by Supabase, make sure the authorized redirect URI is Supabase's OAuth callback URL, not the app's /auth/callback route:

https://<supabase-project-ref>.supabase.co/auth/v1/callback

Supabase receives Google's response there, then redirects the browser to the app callback URL from the allow list.

Phone Install

  1. Open the production site in Safari on iPhone.
  2. Sign in with Google.
  3. Use Share -> Add to Home Screen.
  4. Launch from the home-screen icon and confirm today's toggle persists after closing and reopening.

Smoke Test

Run this before and after deployment:

cd C:\dev\personal\habittracker\web
npm.cmd install
npm.cmd run lint
npm.cmd run build
  1. Visit /login.
  2. Complete Google sign-in.
  3. Confirm the callback returns to /.
  4. Create a habit.
  5. Toggle it for today.
  6. Open the same account on another device/browser and confirm the habit/log appears.
