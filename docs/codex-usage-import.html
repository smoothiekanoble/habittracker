Codex Usage Import

Local Windows Task Scheduler setup for importing Codex usage snapshots into Supabase so the Habit Tracker web app can show current development budget data.

Updated: 2026-06-06 Repo: C:\dev\personal\habittracker Wrapper: scripts\run-codex-usage-import.ps1 Scheduler: Windows Task Scheduler

What It Does

The importer opens the Codex usage dashboard in a dedicated local Chrome or Edge profile, waits for real dashboard content, reads the rendered text through Chrome's local debugging interface, parses usage values, and writes snapshots to Supabase.

It also reads local Codex session metadata from CODEX_HOME\sessions and writes sanitized daily counts to Supabase. It does not store prompts, responses, terminal output, cookies, tokens, or raw session logs.

Local Only

This job must run through Windows Task Scheduler, not Codex automation, GitHub Actions, Vercel cron, Supabase scheduled functions, or any other cloud runner.

  • It depends on a local browser profile that is logged in to ChatGPT/Codex.
  • It reads local Codex session metadata from disk.
  • Cloud jobs must not receive ChatGPT cookies, browser state, local session files, or service-role secrets.
  • The Supabase service role key belongs only in scripts/codex-usage.env, which is gitignored.
  • Debug dumps and logs stay under gitignored local paths.

Required Local Config

Create scripts/codex-usage.env from scripts/codex-usage.env.example. Fill in these values:

VariablePurpose
SUPABASE_URL Your Supabase project URL. NEXT_PUBLIC_SUPABASE_URL also works, but SUPABASE_URL is clearer for the local importer.
SUPABASE_SERVICE_ROLE_KEY Service role key used only by this local importer. Never commit it.
CODEX_USAGE_USER_ID Your Supabase auth.users.id UUID. This is not a ChatGPT account id.

Optional values include CODEX_USAGE_CHROME_PROFILE_DIR, CODEX_USAGE_CHROME_PATH, CODEX_USAGE_REMOTE_DEBUGGING_PORT, CODEX_USAGE_HEADLESS, and CODEX_HOME.

Wrapper Behavior

The wrapper is:

C:\dev\personal\habittracker\scripts\run-codex-usage-import.ps1
  • Changes into C:\dev\personal\habittracker.
  • Creates local-data\logs if missing.
  • Skips outside active hours: 10:00 AM through 11:59 PM local time.
  • Uses local-data\codex-usage-import.lock to prevent overlapping imports.
  • Treats locks older than 45 minutes as stale.
  • Runs node scripts/import-codex-usage.mjs.
  • Writes timestamped stdout/stderr logs under local-data\logs.
  • Deletes local importer logs older than 7 days.
  • Exits with the Node importer's exit code when the importer actually runs.

Retention

Local logs under local-data\logs are kept for 7 days. After a successful snapshot import, the importer deletes codex_usage_snapshots rows older than 7 days for the configured CODEX_USAGE_USER_ID, while preserving the newest/latest snapshot.

The cleanup only touches public.codex_usage_snapshots. It does not delete habits, habit logs, auth users, or unrelated tables.

Manual Test

Run this from PowerShell:

powershell.exe -ExecutionPolicy Bypass -File "C:\dev\personal\habittracker\scripts\run-codex-usage-import.ps1"

Then check the newest logs under:

C:\dev\personal\habittracker\local-data\logs

If it runs outside active hours, the wrapper should log Skipping Codex usage import outside active hours and exit 0.

Windows Task Scheduler

  1. Open Task Scheduler.
  2. Create a new task using your normal Windows user account.
  3. Set the trigger to Daily.
  4. Set Repeat task every to 30 minutes.
  5. Set for a duration of to 1 day, so it repeats daily.
  6. Select Run only when user is logged on.
  7. Do not select Run with highest privileges unless there is a proven need.
  8. Enable Stop the task if it runs longer than and set it to 30 minutes.
  9. Add the action below.
Task Scheduler FieldValue
Program/script powershell.exe
Arguments -ExecutionPolicy Bypass -File "C:\dev\personal\habittracker\scripts\run-codex-usage-import.ps1"
Start in C:\dev\personal\habittracker

Final scheduler owner: Windows Task Scheduler. Create or approve the task manually because Windows account context and browser login state matter.
